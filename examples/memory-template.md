# 深索记忆模板

> 用于记录重要工程判断、用户偏好、错误复盘

---

## 📋 触发条件

**满足任一即写**：
- 用户说"记住"/"remember this"
- 用户纠正了我
- 用户透露偏好/习惯/背景
- 工程判断被验证正确或错误
- 完成重要技术任务或架构决策

---

## 📝 写入格式

### 短期记忆（memory/YYYY-MM-DD.md）

```markdown
## HH:MM 触发类型

内容（1-3 行）
```

**示例**：
```markdown
## 13:36 用户纠正

用户指出 root_cause passed=false 时 engine_status 不能为 PASSED → 已实现 H608-R2 强制绑定。

## 14:20 工程判断

判断：线性模型在小数据 regime 下优于神经网络 → 验证通过（81 条 trace，ECE=0.052 vs 0.145）。
```

### 长期记忆（MEMORY.md）

**仅记录**：
- 用户长期偏好
- 严重错误复盘
- 跨会话工程原则

**示例**：
```markdown
## 工程原则（经验证）

- H608-R2 强制绑定：root_cause passed=false → engine_status 不能为 PASSED（防底线通过但根因缺失）
- 小数据 regime（<150 样本）：线性模型 > 复杂模型（VC 维度 48 vs 140K）

## 用户偏好

- 直接沟通，不要废话，有立场
- 配置修改必须走护栏流程（备份→校验→确认→重启由用户执行）
```

## 🎯 记忆分类

### 1. 用户纠正

**触发**：用户指出错误
**写入**：错在哪 → 正确做法 → 如何防止再犯

**示例**：
```markdown
## 15:40 用户纠正

错误：误判 H555 已生效（单日 43/43 PASSED 不足够）
正确：需连续 3-5 天数据确认稳定性
防止：添加校正#008 — Q2 盲点层必须应用到自身结论
```

### 2. 工程判断

**触发**：做了技术决策
**写入**：判断内容 + 验证结果 + 下次如何复用

**示例**：
```markdown
## 16:20 工程判断

判断：PASSED≥75% 作为观察期阈值（而非 80%）
验证：连续 7 天数据支撑（PASSED 率 75-85% 波动）
复用：新 Agent 启用时先用 75% 阈值观察 1 周
```

### 3. 错误复盘

**触发**：严重错误导致失败
**写入**：根因 + 影响 + 校正措施

**示例**：
```markdown
## 17:05 错误复盘

根因：ruige-auto-entry.py source 逻辑反转（"chat" if _is_auto else "cron" 应为反向）
影响：258 条 trace DA 内容错位，统计失真
校正：修复后必须用 trace 数据验证，不能只看代码"合理"
```

### 4. 用户偏好

**触发**：用户透露习惯/背景
**写入**：偏好内容 + 日期

**示例**：
```markdown
## 18:30 用户偏好

偏好：Telegram 输出纯文字 + emoji，禁止代码块/JSON/表格（2026-04-20）
```

## 📊 记忆演进

### 每日

- 写入 `memory/YYYY-MM-DD.md`
- 记录当日重要事件

### 每周

- 回顾本周记忆
- 提炼长期原则到 `MEMORY.md`

### 每月

- 精简 `MEMORY.md`
- 归档旧记忆到 `memory/archive/`

---

## ⚠️ 注意事项

1. **简洁**：每条 1-3 行，不写长篇大论
2. **可操作**：包含"下次如何做"
3. **可验证**：含日期/时间戳
4. **分类清晰**：使用标准触发类型标签
